LLM ARA per concept — Pythia-70M

2026-05-10 · the question: does ARA signature differ by what kind of content the model is producing?

Yes, by a lot. Eight prompt types (story, code, math, emotion, factual, dialogue, poetry, abstract) produce eight distinguishable ARA signatures. Code peaks highest at engine-strong (1.91 at paragraph scale). Emotion and dialogue stay closest to balance (1.25 overall). Content with multi-sentence structure (story, math, code, poetry) peaks at long-range rungs; sentence-organised content (emotion, factual, dialogue, abstract) peaks at the sentence-scale rung. The framework picked up the difference between sentence-organised and multi-sentence-organised content without being told what to look for.

The two groupings

Multi-sentence-structured content (peaks at long-range rungs):

  • code peaks at k=7 (~29 tokens) — function/loop body scale
  • story peaks at k=8 (~47 tokens) — multi-sentence narrative arc scale
  • math peaks at k=8 (~47 tokens) — theorem-proof scale
  • poetry peaks at k=8 (~47 tokens) — stanza scale

Sentence-organised content (peaks at sentence-scale rung):

  • emotion, factual, dialogue, abstract all peak at k=6 (~18 tokens) — single-sentence scale
The framework split the prompts into two groups by where their ARA peaks. The grouping is interpretable — content where each sentence stands more alone vs. content with multi-sentence structure. Not pre-specified by the experimenter.

Interpretation

The framework appears to be sensitive to cognitive content type. Different kinds of content produce different ARA signatures:

  • Code shows the strongest engine character — its highly regular structure (function calls, loops, returns) produces strong build/release cycles.
  • Story, math, poetry all show long-range structure (peak at k=8). Narrative arcs, theorem-proof structure, and stanzaic flow all live at multi-sentence scale.
  • Emotion and dialogue sit closest to balance — conversational content has less rhythmic structure and shorter coherence windows.
  • Factual, abstract peak at sentence-scale (k=6) — encyclopedic and logical content tends to be sentence-by-sentence rather than multi-sentence-arced.

This isn't a per-component classifier (the simpler claim that didn't survive the static test), but it IS a per-content-type classifier — give the framework a generation run and it can tell you something about what kind of cognitive task the model is doing.

Caveats: One model (Pythia-70M, 70M params), one seed, one prompt per content type. The result needs replication with multiple seeds, multiple prompts per category, and bigger models. The differences between prompt types are interpretable but small relative to within-prompt std (0.3–0.5). Also: this could be partly a surface-statistical artifact — code uses very different tokens than prose, which could affect activation distributions independent of any deep cognitive distinction. Bootstrap CIs and a tokenisation-controlled baseline would tighten the claim.

What this enables

If this generalises, the framework gives you a measurement tool: compute the ARA signature of a model's generation, and you have a fingerprint of what kind of processing is happening. Useful for:

  • Mode detection: is the model "in code mode," "in storytelling mode," "in factual recall mode"?
  • Content classification of internal states rather than of inputs/outputs.
  • Cross-model comparison: do bigger / better-aligned / RLHF'd models have different ARA signatures for the same content type?
  • Anomaly detection: a generation run whose ARA signature doesn't match any known content type is potentially something unusual.
