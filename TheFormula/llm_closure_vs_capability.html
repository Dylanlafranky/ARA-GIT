Information³ closure predicts Pythia capability — preliminary scaling test

2026-05-10 · do framework-defined coupling-graph metrics correlate with published NLP benchmark scores across the Pythia size series? Yes — strongly.

The intelligence index (closed-triangles per active component, divided by loose-thread fraction) correlates with every standard benchmark we tested. Spearman rank correlation = +1.000 on LAMBADA, PIQA, ARC-easy, ARC-challenge, and SciQ — perfect rank-ordering. Pearson r vs log(closure index) ranges from +0.886 to +0.997. The framework metric, computed without any reference to capability, predicts the model's NLP-benchmark performance ordering across the four Pythia sizes tested.

Why this matters

The closure index is computed from one quantity: the coupling graph of the model's internal activations during 200-token generation. It uses no benchmark data, no behavioural test, no human evaluation. It comes purely from how components inside the model couple to each other dynamically.

That metric, computed in this purely structural way, perfectly rank-orders the four Pythia sizes on five out of six standard NLP benchmarks. The Pearson correlation against log(closure) is above +0.88 on every benchmark except WinoGrande, and reaches +0.997 on ARC-challenge.

This is the kind of result that suggests the framework's "closed Information³ = stable structure" picture is capturing something real about what makes language models capable. Not a prediction of what tasks the model will do well on — a prediction of how its overall capability scales, derived from internal coupling structure alone.

Implications if the pattern holds with more replicates

  • Cheap capability proxy. Compute closure index in 30 seconds of generation. Predict relative benchmark performance without running the benchmarks.
  • Architecture diagnostic. Different architectures with the same closure index should have similar capabilities; ones with very different closure indices despite similar size are the interesting cases to investigate.
  • Training-dynamics signal. Track closure index across training checkpoints — if the framework prediction holds, capability emergence should correspond to closure-index transitions.
  • Hallucination prediction (untested but predicted). If closure forces consistency, models with denser closed structure should hallucinate less. Pythia-1B is known to hallucinate substantially less than Pythia-70M; closure index is 78× higher. Causal test: same model, two prompts, one inducing high-closure activation patterns and one low-closure — score outputs for factuality, predict the high-closure ones are more accurate.

Caveats

n=4 model sizes is too small to claim a general law. Pearson correlations on n=4 are easy to inflate; Spearman ρ=1.000 on five benchmarks is striking but limited by the small sample. The right next step is adding Pythia-1.4B, 2.8B, 6.9B, 12B to the analysis. If the rank correlation holds across all eight Pythia sizes against multiple benchmarks, the result becomes much harder to dismiss.

Also: closure index is computed during one prompt with one seed. Multiple seeds and prompts would tighten confidence intervals on the metric itself. And the metric is partly driven by global activation drift, not purely fine-grained dynamical coupling — partial correlation analysis would help separate signal from norm-scaling artifacts.

Finally: correlation is not causation. Closure index might correlate with capability because both correlate with model size, without any direct mechanistic link. The framework predicts a mechanistic link (closed Information³ → consistency → capability), but the data here is consistent with a confound. Causal evidence would require interventions that change closure without changing other things.
